# THIRD_PARTY_LICENSES.md

This file tracks third-party dependency attribution and license compatibility checks.

## Release-Time Workflow

1. Generate a complete dependency list (production and development).
2. Record package name, version, license, source URL, and notice requirements.
3. Verify each dependency license is compatible with repository distribution model.
4. Include required attribution text and notice files in release artifacts.
5. Re-run checks whenever dependencies are added, removed, or upgraded.
